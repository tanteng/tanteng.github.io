Navicat将Sql Server 2000数据库导入（转换）到MySql中

Apr 22, 2014 in DEVELOP • MYSQL • 数据库
1 min read

大家都知道， Sql Server 2000的数据库只能运行在MS系列平台下，而我们有时需要将MS下的数据库移植到Linux或其他平台上，这是可能需要数据库的导入导出功能，但这样也未免太繁琐了，下面介绍一种简便的方法一步实现数据库的移植。

Navicat 8是针对mysql的一个很好用的图形化管理工具，利用它导入功能把数据从sqlserver2000中直接导入到MySQL中。

下面请看步骤：

1.在Navicat中新建一个数据库，打开之后，在table上右键选择“import wizard”。

在弹出的选项卡中选择单选按钮“ODBC”，单击“next”。

在弹出的“数据连接属性”对话框上选择““Microsoft  OLE DB Provider for SQL server”，单击“next”进入下一步

2、在“连接”的卡中输入你的sqlserver2000 的信息，你可以输入你的sqlserver服务器的名称，或者是你想连接的计算机的ip（条件不严格也可以什么都不输，即为默认）,在“输入登陆服务器的信息”中，如果选择第二个单选按钮，需要输入你的sqlserver的名称和密码，然后选择你要导入数据源的数据库名称，单击“测试连接”，如果成功，就可以了，如果没有成功修改其他的选项，要根据错误提示信息来进行修改。

如果没有问题，就可以成功导入了。

3、如果导入有问题，可能的原因有(1):系统防火墙没有关闭；（2）：杀毒软件的防火墙没有关闭；（3）：sqlserver服务器没有正常启动;(4)可以换成系统身份进行连接
