left join多张表查出重复数据的解决办法

May 19, 2015 · 1 minute read
DevelopMySQLPHPThinkPHP数据库

今天线上出现一个问题，查询售后订单出现很多重复的内容，经查，是left join多张表导致的查询出重复数据。

首先看看出现重复数据的时候打印出的sql：

SELECT `bo_id`,`bo_back_no`,`bo_type`,`bo_user_id`,`bo_seller_id`,`bo_order_goods_id`,`bo_order_no`,`bo_order_uname`,`bo_is_received`,`bo_payed`,`bo_money`,`bo_reason`,`bo_refund_note`,`bo_deal_status`,`bo_delivery_code`,`bo_logistics`,`bo_add_time`,`bo_last_modify`,`bo_act_user` 
FROM `js_back_order` 
LEFT JOIN js_sorder_goods as sg on sg_order_no = bo_order_no 
LEFT JOIN js_sorder_info as soi on sg_order_no = soi_no WHERE ( (`bo_add_time` >= '2015-05-12') AND (`bo_add_time` < '2015-05-20') ) AND ( `bo_seller_id` = '19318867' ) 
ORDER BY bo_id DESC LIMIT 0,20

把这条sql在数据库执行，确实出现多条重复的数据。这条sql是把js_back_order作为左表连接的，这张表的bo_order_no是不会有重复数据的，问题出现在第二个left join的时候是以js_sorder_goods作为左表，这张表相同的bo_order_no是可能有多条数据的。

所以解决办法是在sql结尾加上group by bo_id，bo_id是唯一的。

修改后的代码：

$data['data'] = M('BackOrder')->join("js_sorder_goods as sg on sg_order_no = bo_order_no ")
                    ->join("js_sorder_info as soi on sg_order_no = soi_no")
                    ->field($fields)
                    ->where($condition)
                    ->order($order)
                    ->limit($limit)
                    ->group('bo_id')
                    ->select();
