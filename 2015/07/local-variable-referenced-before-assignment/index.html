local variable ‘count’ referenced before assignment

Jul 11, 2015 · 1 minute read
DevelopPython3

在写一个Python闭包函数实现计数器功能的时候，出现了报错：local variable ‘count’ referenced before assignment.这是怎么回事呢？

一开始程序是这样写的：

def hellocounter (name):
    count=0
    def counter():
        count+=1
        print('Hello,',name,',',count,' access!')
    return counter

hello = hellocounter('ma6174')
hello()
hello()
hello()

count变量设置为整数0，闭包函数给count直接加1，这样写是错误的。正确写法是：

def hellocounter (name):
    count=[0]
    def counter():
        count[0]+=1
        print('Hello,',name,',',count[0],' access!')
    return counter

hello = hellocounter('ma6174')
hello()
hello()
hello()

这里就涉及到作用域的问题，前面count变量已经定义了，后面的函数中又使用之前定义过的变量并改变它的值，这样就会报错，就算解决了作用域的问题，给count前加上global或者nonlocal虽然不会报错了，但是没有实现计数功能。

为什么会这样呢？其实是因为count的类型有关，我们知道Pyhton中的基本数据类型分为可变和不可变，

对于不可变类型的赋值，其实是重新定义一个新的变量对象，并深拷贝原对象到新对象，参考str类型说明。

所以这里给count定义一个list类型如[0]的变量,count[0]的值是0，给count[0]累加就没有问题了。
