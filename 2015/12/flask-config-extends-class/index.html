Flask配置文件使用类和继承方式

Dec 11, 2015 in DEVELOP • FLASK • PYTHON3
1 min read

Flask中加载配置文件的方法是：

from flask import Flask, session, g, render_template

from flask_openid.ext.openid import OpenID

app = Flask(name) app.config.from_object(‘websiteconfig’)

如果是第5行代码的方式，那么这里websiteconfig是一个py文件(websiteconfig.py)，里面用大小定义配置信息，如：

DEBUG = False

SECRET_KEY = ‘testkey’

现在介绍另一种更优雅的方式在Flask中定义配置文件。

网站往往至少有两个开发环境，本地测试环境，和线上生产环境，可以先定义一个生成环境的配置，再定义一个测试环境的配置，测试环境的配置可以覆盖生产环境的，使用类和继承的方式定义配置文件是一个很好的方式。

如定义一个配置文件websiteconfig.py：

# coding=utf-8 import os

class Config(object): _basedir = os.path.abspath(os.path.dirname(file))

DEBUG = False

SECRET_KEY = 'testkey'
DATABASE_CONNECT_OPTIONS = {}
ADMINS = frozenset(['http://pythonflask.sinaapp.com/'])

class TestingConfig(Config): BLOG = ‘https://blog.tanteng.me'

del os

测试环境配置是TestingConfig这个类定义的常量，它继承Config类，同时添加了一个配置信息。

引入不同的配置信息的方法，只需要修改app.config.from_object方法的参数，如要使用TestingConfig的配置：

app.config.from_object(‘websiteconfig.TestingConfig’)

这样就导入了测试环境的配置。

更多参考：http://www.pythondoc.com/flask/config.html
