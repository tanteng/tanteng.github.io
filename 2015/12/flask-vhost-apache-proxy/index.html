Flask配置本地虚拟主机访问及Apache反向代理

Dec 11, 2015 · 1 minute read
DevelopFlaskPython3

本地访问Flask应用，首先要启动应用，如下：

(venv)tantengdeMacBook-Pro:pythonflask tanteng$ python run.py
 * Restarting with stat
 * Debugger is active!
 * Debugger pin code: 200-154-157

这样表示应用开始运行了，Flask自带Web Server服务，默认监听端口5000，只需要在浏览器访问127.0.0.1:5000就可以了。

但是如何通过虚拟主机访问呢？比如本地hosts绑定一个虚拟域名www.learn-flask.com来访问本地应用。

在PHP中，配置apache的vhost就可以实现，但是在Python中如何实现？

它要通过一个叫做wsgi的东西来配置，由于本机是Mac OSX系统，安装了一个PHP的集成环境MAMP，自带的Apache也默认装了mod_wsgi模块，以及后面需要用到的反向代理的几个模块。但是Windows下的许多集成环境默认好像是不带的，需要自己去下载或编译。

那就使用这个MAMP带的Apache服务器吧，httpd.conf文件不用修改（视自己环境而定，没装模块的需要先装mod_wsgi），在httpd-vhost.conf中添加如下配置：

<VirtualHost *:80>
    ServerName www.learn-flask.com

    WSGIDaemonProcess learn-flask threads=5
    WSGIScriptAlias / /Users/tanteng/pythonflask/index_loacl.wsgi
    ProxyRequests Off

    <Directory /Users/tanteng/pythonflask>
        WSGIProcessGroup flask
        WSGIApplicationGroup %{GLOBAL}
        Order deny,allow
        Allow from all
    </Directory>
    ProxyPass / http://127.0.0.1:8080/
    ProxyPassReverse / http://127.0.0.1:8080/
</VirtualHost>

重启Apache，通过浏览器访问www.learn-flask.com:8080即可访问了，但是，这样还是不够好，我想去掉端口直接访问网址却不行了，那么就需要设置Apache反向代理。

Flask还需要一个wsgi服务实现Apache的请求，以上配置中的WSGIScriptAlias指定了一个index_local.wsgi文件，内容如下： 

import sys
sys.path.insert(0, '/Users/tanteng/pythonflask/')

from run import app as application

py的入口文件，比如run.py：

from website import app

if __name__ == '__main__':
    app.run(debug=True,port=8080)

注意使用wsgi协议访问必须把app.run写在if __name__ == ‘__main__’里，避免每次请求都创建实例。

设置Apache反向代理

简单说下什么是Apache反向代理，设置反向代理后当你访问www.learn-flask.com实际上访问的就是http://127.0.0.1:8080/，它会自动访问指定端口。至于原理我也不清楚，对Apache服务器感兴趣可以深入研究，这里讲讲设置的方法，实现想要的效果。

这里注意几个地方，千万不能错：

  • 这里必须是80端口，无论你Python应用设置什么端口，因为这里Apache默认是监听的80端口

  • 设置Apache反向代理增加的几个设置项为ProxyRequests,ProxyPass,ProxyPassReverse

  • Proxy,ProxyPasReverse的网址必须以’/’结尾，否则无法正确转发请求

以上配置根据自己的情况修改，把路径部分替换成自己的。

这里只是介绍在Apache服务器中运行的方式，还可以用Nginx，独立wsgi容器等其他方式。
