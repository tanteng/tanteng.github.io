SAE使用第三方Python依赖包

Sat, Dec 12, 2015

在SAE上创建了一个Python Flask应用，可以正常运行，但是在本地安装测试了Flask-Cache模块后传上去，网站无法运行，提示缺少包，如何在SAE使用第三方Python依赖包呢？

把本地运行环境下venv/Lib/sete-packages目录复制一份到根目录，把这个目录通过git或svn传到SAE，也就是说在SAE上全部使用我们本地的包，不使用SAE自带的依赖包。

传上去之后需要修改index.wsgi文件如下：

import sae
sae.add_vendor_dir('site-packages')

from website import app
application = sae.create_wsgi_app(app)

使用sae模块的add_vendor_dir方法，指定使用我们传上去的包目录。注意代码的顺序跟上面一样。
