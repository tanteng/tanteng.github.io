.git/config文件泄露修复

Apr 4, 2016 in 服务器
1 min read

阿里云的云盾系统报告一个系统漏洞，提示：系统配置泄露，路径是https://blog.tanteng.me:80/.git/config，用浏览器打开，一个文本文件就下载下来了，内容如下（经修改）：

[core] repositoryformatversion = 0 filemode = true bare = false logallrefupdates = true [remote “origin”] url = xxxxx fetch = xxxxxx [branch “master”] remote = origin merge = refs/heads/master

乌云安全平台上有多个类似的漏洞，如http://www.wooyun.org/bugs/wooyun-2014-076372

黑客利用这个配置文件可以扫描网站的所有目录，甚至可以还原git版本，还是比较可怕的，赶紧修复这个安全问题！

使用Nginx配置禁止外部访问.git目录

打开网站Nginx配置文件，添加如下配置：

location ~ /.git { deny all; }

这样就禁止从外部访问.git目录，防止git配置文件泄露，再次打开显示403 Forbidden，这样就表示OK了。
