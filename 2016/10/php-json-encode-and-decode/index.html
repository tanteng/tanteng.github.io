PHP json_encode 数组键是否从0开始并且连续的区别

Oct 28, 2016 in DEVELOP • PHP
1 min read

对以下两个数组进行 json_encode 编码，这两个数组的区别是，一个键是从0开始，并且连续，另一个键不连续，那么打印的结果有什么不同呢？

print_r(json_encode([ 0 => 5, 1 => 4, 2 => 1 ]));

print_r(json_encode([ 0 => 5, 1 => 4, 5 => 1 ]));

打印结果：

[5,4,1]

{“0″:5,”1″:4,”5”:1}

第一个是数组的形式，第二个是对象的形式。

如果键不从0开始，或者键不是升序连续的，那么 json_encode 的结果就是键值对的形式。

再看看 json_decode 对于数组或者对象的解析。

例子：

var_dump(array_values(json_decode(”[3,4]“,true)));

var_dump(array_values(json_decode(‘{“1”:3,“2”:4}‘,true)));

执行结果都是：

array(2) { [0]=> int(3) [1]=> int(4) }

array(2) { [0]=> int(3) [1]=> int(4) }

无论是数组还是对象，经过 json_decode 之后都变成了数组，再用 array_values 取值所以结果相同。
