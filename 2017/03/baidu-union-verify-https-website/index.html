百度联盟如何验证 HTTPS 站点

Mar 22, 2017 · 1 minute read
互联网

本站是 HTTPS 站点，在百度联盟添加站点页面，输入 blog.tanteng.me 点击“验证”进入验证页面，提示两种方式，1. 把文件放根目录 2. 添加 meta 信息，都试了，无法验证。

[root@iZ94r80gdghZ public]# curl blog.tanteng.me
<html>
<head><title>301 Moved Permanently</title></head>
<body bgcolor="white">
<center><h1>301 Moved Permanently</h1></center>
<hr><center>nginx/1.10.1</center>
</body>
</html>

原因很简单，本站做了对 HTTP 请求的 301 跳转，使用 curl 命令访问站点就知道返回的内容，如上所示，因此百度联盟验证的时候请求不到需要验证的内容，这个是百度联盟验证机制的不灵活导致的。

解决办法是：

在 nginx 配置中，把 blog.tanteng.me 的配置，由：

server {
    listen       80;
    server_name  blog.tanteng.me;
    return 301 https://blog.tanteng.me$request_uri;
}

临时改成：

server {
    listen       80;
    server_name  blog.tanteng.me;
    root   /html/blog.tanteng.me;
    index  index.php index.html index.htm;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }
    
    #return 301 https://blog.tanteng.me$request_uri;
}

也就是放开 HTTP 80 端口的请求，保证 HTTP 正常访问，保存 nginx 配置并重启，再次按照流程验证即可通过，通过之后再把配置改回去。

相信很多人也会遇到这个问题，而很多站长可能不知道网站配置的知识，所以记录一下，提供参考。
