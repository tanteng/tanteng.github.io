WEB API接口安全和防刷

Jun 16, 2017 in DEVELOP • PHP
1 min read

做各种活动经常会涉及到送会员、礼券、奖品、抽奖、刷榜等业务，这些活动都有一个共同点，那就是恶意用户找到漏洞请求接口从而获取利益，所以要做好这些WEB接口的安全，这里总结一下实践中对这些WEB接口做的安全措施和防刷机制。

1.提高参与门槛

比如至少要求用户登录，要达到一定条件，完成一些任务等等，才能领取奖品，抽奖。同时，这些完成任务的接口也要做到没有漏洞。

2.设定发放礼物的限制

比如奖品的每天总量限制，全部总量的限制，每个用户参与次数限制，每个用户领取总量的限制等等。

3.IP频率限制

通过 memcached 和 redis 都有成熟的方案。

4.验证码&短信验证码

对于一些关键操作可以要求用户输入验证码，或者通过短信验证码的方式进行验证，这可以有效用来接口的防刷，但是这种方式也不一定完美，现在有很多黑卡手机号可以用来获取验证码等，要屏蔽这种黑卡手机号。同时也要保证短信接口不要被刷。

5.用户权限判断

基础的用户行为分析，就是结合业务逻辑，在代码实现层面对进行完善的用户权限判断。

6.防范XSS、CSRF、SQL注入攻击

这是常见的WEB接口安全防范手段，XSS、CSRF、SQL注入，对参数过滤转义，表单验证等等。

7.人工干预

以上方法需要结合使用才能提高安全和破解的难度，单独使用都有很容易破解的方法，除此之外，人工干预也是很重要的，一定要在后台观察数据，及时发现异常的数据并排查隐患。
