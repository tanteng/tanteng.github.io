WEB开发——用户认证模块

May 21, 2018 · 1 minute read
Develop

一个 WEB 项目，用户认证模块几乎是必不可少的，除非不需要用户登陆使用，用户认证的方式主要有基于 Basic 的认证，基于 session 的方案，JWT 方案等。以传统的 session 方案说明，无论哪种方式，它至少包含下列的几个方法：

  • 注册（严格说不属于用户认证模块）
  • 登录
  • 退出
  • 重置密码
  • 基本权限（权限管理也可以和用户认证分开）

除此之外，还可以封装很多其他的方法作为补充，以方便开发。如：

  • 获取当前登陆用户
  • 当前用户是否访客
  • 当前认证用户的ID

还有很多，做的完善一点，还可以实现诸如“一次性登陆”，“记住我”等等。

本文简单介绍一下一些功能的实现。

用户注册

用户注册的流程是：

  1. 首先验证用户名、密码，除此通常还有验证码等进行合法性校验，如用户名是否可用，密码复杂度，验证码是否匹配等。
  2. 创建一个用户，密码采用 password_hash 进行加密存储。
  3. 执行登陆操作并重定向首页（或其他页）

用户登陆

用户登陆的流程是：

  1. 验证用户名和密码输入格式，判断用户尝试次数
  2. 尝试登陆，即验证用户名和密码是否匹配。如果登陆失败，尝试次数+1等。
  3. 如果登陆成功，更新用户 session 信息。

判断用户登录态

判断用户登录态，就是尝试获取登陆用户，如果不存在，即未登录。

从 session 中取出用户 userid，再从数据库中查询并返回。这个方法可能在一次请求中多次调用，可以判断如果存在登陆用户就直接返回，避免重复查询数据库。

用户登出

  1. 获取用户信息
  2. 从 session 中清除这个用户内容
  3. 更新类的用户、是否登出标志等属性
